What's this?

Replies between users flow through the system, and we try to find ones which might be spam.

This page shows the replies which you need to review. Until you do, they won't be visible to the recipient (i.e. the user it was sent to).

  • If the message is fine, click 'Not Spam'. It'll go through and the recipient will see it or receive it by email. If you get a lot of messages from the same person which are trapped, e.g. because of a signature, then click 'Release and Whitelist' which will put this message through and not hold any future messages from this sender.
  • If the message is Spam, click 'Spam'. Then it will not be visible to the recipient. The sender will still see it, so that they don't try different ways to get through. If you do this to a reply, it won't affect the original message.
  • If the message is not Spam, but it's against the rules, then you can choose:
    • Click 'Delete'. The message won't go through. This won't affect the original message.
    • Click 'Add Mod Message'. Explain what's wrong; both sides will see this. Then click 'Not Spam' to let the message through..

Legal Stuff

There are some things you need to be aware of:

  • You are checking these messages for the purpose of detecting spam.
  • In checking these messages, you are acting on behalf of Freegle.
  • You must not disclose any of these messages to any third party - either an individual or another organisation.
  • If you see messages which you think relate to criminal activity (e.g. fraud) then:
    • You may report your concerns within Freegle to support@ilovefreegle.org. Please include the number of the messages shown in grey on the top right and an explanation of your concern.
    • Please then mark them as spam to ensure that the member does not see them.
  • If you see a message which is clearly against Freegle rules (e.g. selling) then you are legally allowed to take further action. We warn users about this in our privacy statement.
  • It is not our business to reject messages which we think are rude. If a member thinks that, they can report the conversation to us.

As long as you follow the procedure above, you are not liable for any issues. For those who are interested in the legalities, this constitutes an authorised interception under section 3.3 of RIPA, and we have agreed this approach with the Interception of Communications Commissioner's Office.
